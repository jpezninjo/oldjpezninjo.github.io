Projects

Search Engine

For my Software Development class, I wrote a search engine application broken up into 5 smaller parts.
  1. Project1 Inverted Index
    • Created an inverted index that mapped word entries to file locations.
    • Given a directory, parse all nested text files and add entries to index.
    • Write index to a json file
  2. Project2 Partial Search
    • Adding onto part 1, given a query of words seperately by whitespace, return all matches from index and output to a json file.
    • Matches were determined by starting with any of the query words, ex. "a" would yield all words beginning with a.
    • Matches were then ranked based on number of locations.
  3. Project3 Multithreading
    • This part added multithreading functionality to our existing framework.
    • Our classes for parsing text files in proj1 and looking up queries in proj2 were extended to support using Java's own thread class to make our whole project more efficient.
  4. Project4 Web Crawler
    • In preparation for proj5, project 4 added the ability to use web pages instead of local text files.
    • Using sockets, we are able to retrieve the whole html of any web address and then strip it clean of html tags.
    • Additionally, our application would alslso parse any web addresses found within the html (multithreaded, of course).
  5. Project5 Search Engine
    • Finally, the last part used the Jetty library (link here) to launch our application onto local host. This part was essentially the creation of a front-end for our first four projects.
